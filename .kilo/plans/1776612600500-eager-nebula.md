# Plan: Documentación de Endpoints para Postman

## Objetivo

Crear documentación de los endpoints de autenticación para pruebas en Postman

## Endpoints a documentar

### 1. POST /auth/login

- **URL:** `http://localhost:3000/auth/login`
- **Body:**

```json
{
  "email": "sam94c@gmail.com",
  "password": "Abc123.."
}
```

- **Respuesta exitosa:**

```json
{
  "data": {
    "access_token": "eyJ...",
    "firstName": "Master",
    "lastName": "Admin",
    "userType": "admin"
  },
  "success": true
}
```

### 2. POST /auth/create-user

- **URL:** `http://localhost:3000/auth/create-user`
- **Headers:** `Authorization: Bearer {access_token}`
- **Body:**

```json
{
  "firstName": "Juan",
  "lastName": "Pérez",
  "email": "juan.perez@lalora.com",
  "phoneNumber": "+50512345678",
  "userType": "waiter"
}
```

- **userType válidos:** admin, kitchen, cashier, waiter,cancha_manager
- **Respuesta:** Devuelve el usuario creado

### 3. POST /auth/recover-password

- **URL:** `http://localhost:3000/auth/recover-password`
- **Body:**

```json
{
  "email": "sam94c@gmail.com"
}
```

### 4. POST /auth/reset-password

- **URL:** `http://localhost:3000/auth/reset-password`
- **Body:**

```json
{
  "email": "sam94c@gmail.com",
  "code": "123456",
  "newPassword": "NuevaPass123"
}
```

## Swagger UI

Disponible en: `http://localhost:3000/api`

```json
      "currency": "COP"
    },
    "accountClosings": [

        },
        {
          "id": "user-3",

          "lastName": "Ruiz",
          "role": "KITCHEN"
        }
      ],
      "totalUsers": 8
    }
  }

```

### Cálculos Requeridos

1. **dailySales**
   - todayTotalSales: Sum de `Order.total` donde status = DELIVERED/CLOSED y fecha = hoy
   - todayOrdersCount: Conteo de órdenes con status = DELIVERED/CLOSED y fecha = hoy
   - Horario Colombia: GTM-5 (si el server está en UTC, restar 5 horas)

2. **accountClosings**
   - Últimos 3 días previos al actual
   - Cada día: totalIncome (sum Orders), totalExpenses (sum Expenses), balance (income - expenses)

3. **menu**
   - categoriesCount: SELECT COUNT(\*) FROM Category WHERE active = true
   - productsCount: SELECT COUNT(\*) FROM Product WHERE available = true

4. **tables**
   - spaces: List de zonas con tablesCount
   - totalTables: COUNT(\*) FROM Table

5. **users**
   - recentUsers: Últimos 3 usuarios creados (ORDER BY createdAt DESC)
   - totalUsers: COUNT(\*) FROM User WHERE active = true

##IMPLEMENTACIÓN: Espacios, Mesas y Menú

### Paso 1: Modelos Prisma (prisma/schema.prisma)

```prisma
// Zone (Espacios)
model Zone {
  id        String   @id @default(uuid())
  name     String
  icon     String   @default("restaurant")
  tables   Table[]
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

// Table (Mesas)
enum TableStatus {
  AVAILABLE
  OCCUPIED
  RESERVED
}

model Table {
  id        String      @id @default(uuid())
  name     String
  zoneId   String
  zone     Zone       @relation(fields: [zoneId], references: [id])
  status   TableStatus @default(AVAILABLE)
  orders   Order[]
  createdAt DateTime   @default(now())
  updatedAt DateTime   @updatedAt
}

// Category (Categorías del menú)
model Category {
  id           String    @id @default(uuid())
  name         String
  description  String?
  icon         String    @default("restaurant")
  displayOrder Int       @default(0)
  active      Boolean   @default(true)
  products    Product[]
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
}

// Product (Productos del menú)
model Product {
  id          String   @id @default(uuid())
  name       String
  description String?
  price      Float
  image      String?
  available  Boolean  @default(true)
  categoryId String
  category   Category @relation(fields: [categoryId], references: [id])
  modifiers ProductModifier[]
  orderItems OrderItem[]
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

// ProductModifier (Modificadores de producto)
model ProductModifier {
  id         String   @id @default(uuid())
  name       String
  options    String[]
  required  Boolean  @default(false)
  multiple  Boolean  @default(false)
  productId String
  product  Product  @relation(fields: [productId], references: [id])
  createdAt DateTime @default(now())
}

// Order (para referencia futura)
enum OrderStatus {
  PENDING
  CONFIRMED
  IN_PREPARATION
  READY
  DELIVERED
  CANCELLED
  CLOSED
}

enum OrderType {
  LOCAL
  TAKEOUT
}

model Order {
  id          String     @id @default(uuid())
  tableId     String?
  table      Table?     @relation(fields: [tableId], references: [id])
  userId     String
  status     OrderStatus @default(PENDING)
  orderType  OrderType  @default(LOCAL)
  total      Float      @default(0)
  items      OrderItem[]
  createdAt  DateTime   @default(now())
  updatedAt  DateTime   @updatedAt
}

model OrderItem {
  id         String  @id @default(uuid())
  orderId    String
  order     Order   @relation(fields: [orderId], references: [id])
  productId  String
  product   Product @relation(fields: [productId], references: [id])
  quantity  Int     @default(1)
  price     Float
  notes     String?
  createdAt DateTime @default(now())
}
```
