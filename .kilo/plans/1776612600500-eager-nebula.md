# Plan: Documentación de Endpoints para Postman

## Objetivo

Crear documentación de los endpoints de autenticación para pruebas en Postman

## Endpoints a documentar

### 1. POST /auth/login

- **URL:** `http://localhost:3000/auth/login`
- **Body:**

```json
{
  "email": "sam94c@gmail.com",
  "password": "Abc123.."
}
```

- **Respuesta exitosa:**

```json
{
  "data": {
    "access_token": "eyJ...",
    "firstName": "Master",
    "lastName": "Admin",
    "userType": "admin"
  },
  "success": true
}
```

### 2. POST /auth/create-user

- **URL:** `http://localhost:3000/auth/create-user`
- **Headers:** `Authorization: Bearer {access_token}`
- **Body:**

```json
{
  "firstName": "Juan",
  "lastName": "Pérez",
  "email": "juan.perez@lalora.com",
  "phoneNumber": "+50512345678",
  "userType": "waiter"
}
```

- **userType válidos:** admin, kitchen, cashier, waiter,cancha_manager
- **Respuesta:** Devuelve el usuario creado

### 3. POST /auth/recover-password

- **URL:** `http://localhost:3000/auth/recover-password`
- **Body:**

```json
{
  "email": "sam94c@gmail.com"
}
```

### 4. POST /auth/reset-password

- **URL:** `http://localhost:3000/auth/reset-password`
- **Body:**

```json
{
  "email": "sam94c@gmail.com",
  "code": "123456",
  "newPassword": "NuevaPass123"
}
```
